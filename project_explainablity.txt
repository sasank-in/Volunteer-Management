1. Project Title: Web Platform for Volunteer Management and Outreach

2. Project Statement and Outcomes:
The project is a web-based application designed to help communities organize volunteer efforts, manage cleanup events, support disaster-affected families, and improve local infrastructure like street lighting. The platform connects volunteers with community projects, tracks participation, and provides real-time updates.

The outcome is a fully functional microservices-based system that enhances community engagement by simplifying volunteer management. Community organizers can post events, track attendance, coordinate volunteers efficiently, and receive feedback. Volunteers can sign up for events, receive real-time notifications, track their participation history, and contribute effectively to community-driven projects. The system includes comprehensive testing, detailed documentation, and production-ready deployment guides.

5. Technical Architecture:

Microservices (Spring Cloud):
- Config Server (8888): Centralized configuration
- Discovery Server (8761): Eureka service registry
- API Gateway (8080): Single entry point with routing
- User Service (8081): Authentication and user management
- Event Service (8082): Event and participation management
- Notification Service (8083): Email and in-app notifications

Technology Stack:
- Java 21
- Spring Boot 3.4.2
- Spring Cloud 2024.0.1
- Spring Security with OAuth2 JWT
- PostgreSQL with Flyway migrations
- Spring Mail for notifications
- Maven for build management

Security:
- JWT-based authentication (HS256)
- Stateless session management
- BCrypt password hashing
- Token refresh mechanism
- Secure token storage (SHA-256)

6. Deployment Options:
- Local development with start.bat script

7. API Endpoints Summary:

Authentication: /api/auth/* (register, login, refresh, logout, password reset)
Users: /api/users/* (profile, list, update, delete)
Events: /api/events/* (create, list, update, delete, filter)
Participations: /api/participations/* (register, cancel, history, attendance)
Feedbacks: /api/feedbacks/* (submit, list, average rating)
Notifications: /api/notifications/* (list, unread, mark read, send)

8. Database Schema:

User Service DB (volunteer_user_db):
- user_accounts: User information and credentials
- refresh_tokens: JWT refresh token storage
- password_reset_tokens: Password reset tokens

Event Service DB (volunteer_event_db):
- events: Event information and status
- participations: Volunteer participation records
- feedbacks: Event ratings and comments

Notification Service DB (volunteer_notification_db):
- notifications: Notification records and delivery status
